Code Organization

    For code organization, the following rules should apply.

  1. A line will separate each major section.
  2. A ine will separate each minor section.
  3. Indentation will be tabs or the equivalent of 3 spaces
  4. Code blocks should have the opening bracket with the name of the declaration and the ending bracket alone on a single line.
  5. Classes will be organized into its relevant categories.

File Naming Conventions

Naming files should follow the following rules

  1. Files with multiple words shall start with a lowercase for the first word and subsequent words will be capitalized on the first letter.
  2. File extensions with alternatives should be chosen with the more commonly used ones such as html over htm, jpg over jpeg.
